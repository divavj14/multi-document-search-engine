Phase 2: Advanced Proctoring Settings – Smart Browser
Smart Browser is a secure desktop application designed to conduct a fair, secure
assessment.
HackerEarth’s Smart Browser is a secure desktop application designed to help you
conduct tests with enhanced proctoring. It offers stricter controls compared to
standard browser-based tests, ensuring a more reliable and cheat-resistant testing
experience.
By using Smart Browser, you can be more confident that candidates are completing
assessments in a controlled and secure environment.
Smart Browser preview does not work for unpublished tests. You need to publish the
test to preview it in the Smart Browser.

Enable Smart Browser in settings
Admins can enable Smart Browser from the proctoring settings while creating or editing
a test. Once enabled, candidates will be required to attempt the test using Smart
Browser only.
How Smart Browser helps prevent cheating?
Smart Browser uses multiple security checks before and during the test to ensure
fairness and integrity.
Security measures before the test
Before the test begins, Smart Browser verifies the candidate’s system setup and
ensures that all security requirements are met. Candidates must launch the test
exclusively through the Smart Browser application.
Smart Browser performs a complete system check to:
•

Detect whether the system is running in a virtual environment

•

Ensure all display requirements are met

•

Verify that the required domains are accessible

•

Check for any open applications

If restricted applications are detected, you can choose to close them manually or allow
Smart Browser to close them automatically.

Security measures during the test
Once the test starts, Smart Browser actively restricts actions that could compromise
test integrity. It prevents candidates from:
1. Sharing or recording the test window
2. Taking screenshots of the test
3. Opening or switching to other applications
4. Resizing the test window
5. Using multiple monitors
6. Running the test inside a virtual machine
7. Accessing browser developer tools

Any attempt by a candidate to move away from the test screen, such as switching
workspaces, exiting full-screen mode, or opening another application in the
background, will trigger a warning.
Candidates cannot open external links and applications while taking an assessment in
the Smart Browser. Question types like Machine Learning and File upload that require
candidates to use a separate application to solve should not be used for assessments
in the Smart Browser.

If the candidate does not return to the test within 10 seconds after the initial warning,
they will be automatically removed from the test.
Restricted keyboard actions
To maintain focus and prevent misuse, the following key combinations are disabled:
•

Function keys and related combinations (for example, F1, Alt + F5)

•

Alt + Tab

•

Ctrl + Alt + Delete

•

Ctrl + C and Ctrl + V

•

Operating system shortcut keys (such as Windows key, Mac Command key,
and combinations like Windows key + C)

Additional controls
•

Operating system notifications are blocked

•

Screen recording and screen-sharing tools are disabled

After the test
After the test is completed, Smart Browser safely exits the secure mode and restores
normal system access.
You can additionally choose to receive the following test reports via email:
•

Email report: Choose to email the assessment summary report to all Test
Admins.

•

Plagiarism report: Receive a detailed plagiarism report after the test is
completed.

•

Detailed assessment report: Include the detailed assessment report (PDF)
along with the summary report.

Before the test
1. After installation, open the Smart Browser.
2. A quick system check is performed. And if everything goes well, you’ll
see a pop-up to enter the test URL.

3. Before starting the test, Smart Browser will prompt you to close all
background applications. You can either close them manually or
choose the option to close all apps automatically.

Note: Some apps may not close even after selecting Try Closing
Automatically. In those cases, you need to close them manually.
During the test
1. Once the test begins, your screen is constantly monitored.
2. Any attempt to move away from the test screen (like switching
workspaces or exiting full screen) or opening another application in
the background will trigger a warning.
Note: You will be removed from the test if you don’t return to the test within
10 seconds after an initial warning.

3. Do not try to record the screen or capture screenshots while
attempting the test.
4. Once you complete the test, the Smart Browser will automatically
close itself.
Important notes
1. If you're using a social login to access the test, make sure to select
the correct account. You won't be able to go back to the previous

screen. To restart the process, you'll need to relaunch the Smart
Browser.
2. If you're disconnected from the test due to a technical issue, you can
resume the test by either:
a. Logging in again through the test invite email, or
b. Relaunching the Smart Browser application directly.
3. Avoid using any external tools such as Interview Coder or other
platforms that provide outside assistance.
4. Do not use multiple monitors while attempting the test. Close any
virtual machine running on your system, as it is not allowed while
attempting the test.
5. We strongly recommend using a stable and reliable internet
connection. Avoid mobile hotspots or tethered networks, as they
may cause interruptions.
6. For questions (Full-stack) where a preview or external link needs to
be opened, it will be opened in a floating window that can be closed
and resized.

Phase 4: Troubleshooting Guide (including Snapshot Disabled
Assessments)
1. Firewall Issues
Candidates may encounter a firewall error due to the following reasons:
•

The machine is connected to a VPN.

•

Google APIs or HackerEarth domains are being blocked by the
network.

•

Candidates should try using a personal laptop instead of an official
one.

•

Alternatively, switching to a different network may resolve the issue.

To turn off firewall follow these steps
Step 1: press win+ r to open run and type control to open control panel.

Step 2: Change view by category to Large Icons on top right of your control
panel window.

Then click on Windows Defender Firewall

Click on Windows Defender Firewall on or off

then click on both marked options to turn of the window firewall.

2. 403 Access Denied Issue
Candidates may see a "403 Access Denied" error when attempting to
access the page. This can occur due to incorrect login credentials.
•

Verify that the login information is correct (email and password).

•

If the issue persists, candidates should clear their browser history
and cache or try accessing the page in incognito mode.

3. Test Loading Issues / Blank Screen / IDE Not Loading
Sometimes, candidates may face loading issues, such as a blank screen or
an IDE not loading properly. These issues are often caused by unstable
internet connectivity.
•

Candidates should try pressing Ctrl + Shift + R (Hard Reload) to
reload all resources from the webpage.

•

We recommend switching to a more stable network to resolve the
issue.

4. Tab Switches
Clicking on any notification or tab during the test can trigger a tab switch,
which may affect the test flow.
•

Candidates are requested to disable notifications before attempting
the test to prevent any interruptions or tab switches.
